1. Was ist die Cortex-Plattform?

Cortex ist ein offenes Bausatz-System für Arztpraxen. Drei Komponenten arbeiten zusammen:

  • Cortex-Web — das offene Framework (Theme, Schemata, Adapter für WordPress, Astro, Shopify). Kostenfrei, Apache 2.0, öffentlich auf github.com/Sanexio/cortex-web — klonen, ansehen, forken, beitragen.
  • Dein Tenant-Repo — deine eigenen Inhalte (Team-Profile, Leistungen, Pages, Impressum). Bleibt unter deiner Kontrolle, ist nicht öffentlich.
  • Cortex-CLI — das Kommandozeilen-Werkzeug, das alles zusammenbringt: Installation, Updates, Tenant-Verbindung.
Wichtig: Du behältst die volle Kontrolle über deine Tenant-Inhalte. Sanexio sieht sie nie — das Lizenz-System überträgt nur den Token, keine Praxis-Daten.

Hinweis: Die Cortex-CLI bringt ihre eigene JavaScript-Runtime (Bun) mit. Du musst weder Node noch Bun manuell installieren.

3. Lizenz erhalten

Vor der ersten Installation brauchst du einen Lizenz-Token im Format XXXX-YYYY-ZZZZ. Den bekommst du nach einer kurzen Identitäts-Prüfung von Sanexio (Dr. Stracke).

Geprüft werden:

  1. Gültige Approbation als Arzt/Ärztin (Kopie der Urkunde reicht, oder Eintrag in der Bundesärztekammer-Datenbank).
  2. Eigene Niederlassung oder Anstellung in einer Praxis/einem MVZ.
  3. Schriftliche Bestätigung der DSGVO-Verantwortung für die eigene Patient-Datenpflege (siehe Kapitel 9).

Schreib für die Lizenz-Anfrage an info [at] sanexio.de mit:

  • vollständiger Name, Facharzt-Bezeichnung
  • Praxis-Anschrift
  • kurze Beschreibung deines Hauptanliegens (öffentliche Webseite, Online-Verkauf von Selbstzahler-Leistungen, beides)

Du bekommst innerhalb weniger Tage deinen Token plus ein eigenes privates Tenant-Repo auf GitHub angelegt.

4. Erst-Installation in einer Zeile

Sobald du den Token hast, öffne dein Terminal und führe diesen Befehl aus:

curl -fsSL https://sanexio.github.io/cortex/install.sh | bash

Das Skript prüft die Plattform, lädt das signierte und Apple-notarisierte Cortex-Binary, verifiziert die SHA-256-Prüfsumme und legt dir den Befehl cortex in ~/bin/ an.

Achtung: Wenn ~/bin/ nicht in deinem $PATH steht, ergänze in ~/.zshrc die Zeile export PATH="$HOME/bin:$PATH" und öffne ein neues Terminal-Fenster. Das Installations-Skript weist dich darauf hin.

Prüfe direkt nach der Installation, ob alles funktioniert:

cortex --version
cortex status

cortex status zeigt dir, dass tenant-dir und license noch fehlen — das ist normal und der Wizard im nächsten Schritt füllt das auf.

5. Geführter Wizard: cortex onboard

Der cortex onboard-Wizard führt dich in fünf Schritten durch das komplette Setup:

cortex onboard

Schritt 1 / 5 — Lizenz-Token

Tippe deinen Token im Format XXXX-YYYY-ZZZZ ein. Der Wizard prüft das Format lokal und meldet sich beim Sanexio-Endpunkt, um den Token zu validieren. Falls du gerade kein Internet hast, läuft der Setup im Offline-Modus weiter und holt die Bestätigung beim nächsten Online-Termin nach.

Schritt 2 / 5 — Tenant-Repo

Gib den Slug deines Tenant-Repos ein, z. B. Sanexio/dr-mustermann-tenant. Der Wizard klont das Repo per HTTPS nach ~/Cortex/projects/<repo-name>. Du kannst diesen Schritt überspringen, falls Sanexio das Repo noch nicht angelegt hat — dann holst du es später per git clone nach.

Tipp: Setze in ~/.zshrc die Zeile export CORTEX_TENANT_DIR="$HOME/Cortex/projects/<repo-name>", damit Cortex-Web deinen Tenant beim nächsten Build sofort findet.

Schritt 3 / 5 — Cortex-Web-Layer

Der Wizard fragt, ob er Cortex-Web nach ~/.cortex/projects/cortex-web klonen soll. Drück Enter für die Default-Antwort (ja), oder n falls du es später nachholst.

Schritt 4 / 5 — LaunchAgent für Auto-Update

Der LaunchAgent prüft einmal täglich im Hintergrund auf Cortex-Updates und zeigt dir eine macOS-Notification an, falls eines verfügbar ist. Anwenden tust du Updates dann manuell mit cortex update --apply. Default ist ja.

Schritt 5 / 5 — Smoke-Test

Der Wizard ruft cortex status auf. Wenn alles grün ist, siehst du Häkchen bei cortex-root, tenant-dir, license und mindestens einem Cortex-Layer-Projekt.

Setup abgeschlossen. Du kannst jetzt deine Inhalte einpflegen und deine Site bauen.

6. Tenant-Inhalte einpflegen

Dein Tenant-Repo startet mit drei anonymisierten Beispiel-Profilen und Schablonen für Impressum und Leistungen. Lösche die Beispiele und ersetze sie durch deine echten Inhalte.

Tenant-Konfiguration anpassen

cd ~/Cortex/projects/<dein-tenant>
mv tenant.config.template.json tenant.config.json
# tenant.config.json öffnen und Werte anpassen:
# - praxis.site_url: deine echte Praxis-URL
# - shop.public_domain: dein Shop-Domain (falls Shop)
# - cta.allowed_external_cta_domains: Whitelist für externe Links

Team-Profile

Lege pro Arzt/Ärztin eine YAML-Datei unter trunk/content/team/ an. Pflichtfelder:

  • id, slug, order — Identifikation und Reihenfolge
  • name, role.de — Name und Funktion
  • languages — Liste der Sprachen, z. B. [de, en, fr]
  • intro.de, bio.de — Vorstellungstexte

Schau dir trunk/content/team/dr-muster.yaml als Vorbild an — alle Pflichtfelder sind dort sauber dokumentiert.

Pages und Impressum

Die DSGVO-Pflicht-Pages liegen unter trunk/content/pages/_shared/. Ersetze die Beispiel-Texte in impressum.yaml und leistungen.yaml durch deine eigenen Angaben.

Niemals Patient-Daten, QM-Handbücher oder Login-Credentials in dieses Repo legen. Die mitgelieferte .gitignore hat ein Sicherheits-Netz dafür eingebaut, aber das ersetzt nicht die Aufmerksamkeit beim Einchecken.

7. Deine Site bauen

Cortex-Web hat drei Adapter, je nach Site-Typ:

AdapterBefehlAusgabe
WordPressbun adapters/wordpress/build-team.mjs
bun adapters/wordpress/build-page-hub.mjs		PHP-Files für ein WordPress-Theme
Astrobash tools/build-astro.shstatisches HTML/CSS
Shopifybun adapters/shopify/build.mjsLiquid-Templates für ein Shopify-Theme

Alle Befehle werden im Cortex-Web-Verzeichnis ausgeführt (cd ~/.cortex/projects/cortex-web) und lesen deinen Tenant über die CORTEX_TENANT_DIR-Variable.

Erstes Mal: Bun-Abhängigkeiten installieren

cd ~/.cortex/projects/cortex-web
bun install

Das richtet die JavaScript-Abhängigkeiten ein, die die Adapter brauchen. Das ist nur einmal pro Mac nötig.

Schema-Prüfung vor jedem Build

bash tools/validate.sh

Prüft alle deine YAML-Dateien gegen die Cortex-Web-Schemata. Wenn etwas fehlt oder das Format falsch ist, sagt dir validate.sh genau in welcher Datei und Zeile.

8. Updates und Pflege

Cortex bekommt regelmäßig Updates — manchmal Bug-Fixes, manchmal neue Adapter oder Schema-Erweiterungen.

Manuell prüfen

cortex update --check

Zeigt dir, ob ein neueres Cortex-CLI verfügbar ist und was sich geändert hat.

Anwenden

cortex update --apply

Lädt das neue Bundle, prüft die Apple-Signatur und ersetzt das alte Binary atomar.

Auto-Update (Notification)

Wenn du den LaunchAgent in Schritt 4 des Wizards installiert hast, prüft Cortex einmal täglich im Hintergrund. Bei einem verfügbaren Update bekommst du eine macOS-Notification — das Apply selbst startest du manuell, damit du den Zeitpunkt kontrollierst.

Tenant-Repo-Updates

Dein Tenant-Repo pflegst du selbst per git pull / git push. Bei Strukturänderungen in Cortex-Web (z. B. neue Pflichtfelder im Schema) bekommst du im Sanexio-Release-Newsletter die Migrations-Anleitung.

9. DSGVO und Patientendaten

Cortex ist ausschließlich für öffentlich-zeigbares Material deiner Praxis gedacht. Patient-Daten gehören in dein praxiseigenes System (PVS, Doctor-Workflow, etc.) und niemals in das Tenant-Repo.

Was darf in dein Tenant-Repo:

  • Team-Profile (Name, Funktion, Lebenslauf, Foto)
  • Leistungsbeschreibungen, Sprechzeiten, Standort-Infos
  • Impressum und Datenschutzerklärung
  • Selbstzahler-Leistungen / Bluttests / Online-Verkaufs-Produkte

Was darf niemals in dein Tenant-Repo:

  • Patient-Akten, Befunde, Diagnosen, Termine
  • QM-Handbuch, Arbeitsverträge, Steuer-Belege
  • .env-Dateien, API-Keys, Passwörter

DSGVO-Verantwortung

Sanexio verarbeitet keine Patient-Daten und benötigt daher keinen Auftragsverarbeitungsvertrag. Du als Tenant bleibst alleiniger Datenschutz-Verantwortlicher für alles, was über die offene Webseite öffentlich wird (z. B. Impressum gemäß § 5 TMG, Datenschutzerklärung gemäß Art. 13 DSGVO).

Empfehlung: Lass den Datenschutz-Text deiner Tenant-Site einmalig von einer Fachjuristin / einem Fachjuristen oder deiner Berufshaftpflicht-Versicherung gegenlesen, bevor du live gehst.

10. Häufige Fehler und Diagnose

cortex: command not found

Dein ~/bin/ steht nicht im $PATH. Lösung: in ~/.zshrc die Zeile export PATH="$HOME/bin:$PATH" ergänzen und ein neues Terminal-Fenster öffnen.

Wizard bricht mit cortex init: Lizenz vom Server abgelehnt — unknown_token ab

Der Token wurde noch nicht auf der Sanexio-Server-Seite eingetragen oder du hast einen Tippfehler. Doppelt prüfen, oder bei Sanexio melden.

cortex add cortex-web meldet Host key verification failed

Du nutzt versehentlich die SSH-Variante ohne hinterlegten GitHub-Key. Lösung: einfach noch einmal aufrufen, der Default ist heute HTTPS und sollte ohne Auth funktionieren.

validate.sh sagt node_modules missing — run bun install

Die Cortex-Web-Abhängigkeiten sind noch nicht installiert. Lösung: cd ~/.cortex/projects/cortex-web && bun install.

nexus-gateway startet mit License-Check FAIL: Keine Lizenz unter ~/.cortex/license.json

Du hast Cortex installiert, aber cortex init --license noch nicht ausgeführt. Lösung: cortex onboard oder direkt cortex init --license XXXX-YYYY-ZZZZ.

Build-Adapter meldet error: Module not found "adapters/wordpress/build-team.mjs"

Du bist im falschen Verzeichnis. Lösung: cd ~/.cortex/projects/cortex-web und Befehl noch einmal ausführen.

macOS sagt "cortex" kann nicht geöffnet werden, da es von einem nicht verifizierten Entwickler stammt

Sollte nicht passieren — die Binaries sind Apple-notarisiert. Falls doch: xattr -d com.apple.quarantine ~/bin/cortex löscht das Quarantine-Flag manuell.

11. FAQ

Wie viel kostet Cortex?

Das hängt vom Modell ab, das Sanexio dir vorschlägt. Cortex-Web (Framework) und Cortex-CLI (Werkzeug) sind kostenfrei Apache 2.0. Die Lizenz deckt Stracke's Support, Sanexio-seitiges Hosting des Lizenz-Endpunkts und Cortex-Update-Pflege. Konkrete Preise im individuellen Gespräch.

Was passiert, wenn ich Cortex aufkündige?

Deine Tenant-Inhalte gehören dir und liegen in deinem eigenen Repo. Du nimmst sie mit. Die Lizenz wird auf Sanexio-Seite revoked, Cortex-CLI verweigert dann den Start. Cortex-Web (Framework) kannst du theoretisch weiter alleine pflegen — ohne Sanexio-Updates allerdings.

Kann ich Cortex auf mehreren Macs nutzen?

Ja, eine Lizenz pro Praxis, beliebig viele Macs (Praxis-Rechner, Mobil-Mac, etc.). Voraussetzung: jeder Mac führt cortex install + cortex init --license <token> aus.

Kann ich Cortex selbst weiterentwickeln?

Cortex-Web ist Apache 2.0 — Pull Requests willkommen unter github.com/Sanexio/cortex-web. Bei Cortex-CLI gilt vorerst, dass nur Sanexio offizielle Releases signiert und notarisiert; du kannst aber deinen eigenen Fork bauen, wenn du willst.
